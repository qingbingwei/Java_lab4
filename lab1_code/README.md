# 学生成绩管理系统

## 项目简介

这是一个基于命令行的学生成绩管理系统，使用Java语言实现。系统能够管理学生、教师、课程、教学班和成绩等信息，支持成绩录入、查询、统计和分析等功能。

## 系统功能

### 核心功能
1. **数据初始化** - 自动生成学生、教师、课程、教学班等基础数据
2. **学生选课** - 智能选课系统，支持自定义选课策略
3. **成绩管理** - 分阶段生成平时、期中、实验、期末成绩
4. **成绩查询** - 支持按学号、姓名查询学生成绩
5. **成绩统计** - 提供多维度的成绩统计分析
6. **成绩排名** - 支持按学号、各科成绩和总成绩排序

### 数据要求
- **学生数量**: ≥100人
- **教师数量**: ≥6人
- **课程数量**: ≥3门
- **教学班**: 每门课至少2个教学班(2个教师)
- **班级规模**: 每个教学班≥20人
- **选课要求**: 每个学生至少选择3门课程

### 成绩组成
成绩由4部分构成:
- 平时成绩 (20%)
- 期中成绩 (20%)
- 实验成绩 (20%)
- 期末成绩 (40%)

**综合成绩计算公式**: 
```
综合成绩 = 平时成绩 × 0.2 + 期中成绩 × 0.2 + 实验成绩 × 0.2 + 期末成绩 × 0.4
```

## 设计特点

### 设计模式
1. **单例模式** (Singleton Pattern) - DataManager类确保全局只有一个数据管理实例
2. **工厂模式** (Factory Pattern) - DataFactory类负责创建各种数据对象
3. **策略模式** (Strategy Pattern) - 实现灵活的成绩生成和选课策略
4. **命令模式** (Command Pattern) - 封装菜单操作为命令对象

### Java高级特性
- **Lambda表达式** - 用于简化函数式接口的实现
- **Stream API** - 用于数据的过滤、映射、排序等操作
- **泛型** - 提供类型安全的集合操作
- **枚举类型** - 管理性别、分数段等常量数据
- **接口与抽象** - 定义统一的行为规范
- **注解** - `@FunctionalInterface`等注解
- **集合框架** - 使用List、Map等集合类

### 面向对象特性
- **封装** - 所有属性私有化，提供getter/setter方法
- **继承** - 可扩展的类层次结构
- **多态** - 接口和抽象类的多态实现
- **关联关系** - 类之间的合理关联设计

## 使用指南

### 系统菜单
```
==============================================================
  学生成绩管理系统
==============================================================
  [1] 初始化数据(生成学生、教师、课程、教学班)
  [2] 学生选课
  [3] 生成平时成绩
  [4] 生成期中成绩
  [5] 生成实验成绩
  [6] 生成期末成绩并计算综合成绩
  [7] 显示教学班成绩
  [8] 查询学生成绩
  [9] 成绩统计分析
  [10] 显示所有学生排名
  [0] 退出系统
==============================================================
```

### 操作流程
1. **初始化数据** - 首次运行必须先初始化数据
2. **学生选课** - 为学生分配课程
3. **生成成绩** - 按顺序生成各项成绩
   - 平时成绩
   - 期中成绩
   - 实验成绩
   - 期末成绩(自动计算综合成绩)
4. **查询统计** - 查看成绩、统计分析、排名等

### 成绩生成策略
- **平时成绩**: 正态分布，平均分75，标准差10
- **期中成绩**: 正态分布，平均分70，标准差12
- **实验成绩**: 正态分布，平均分80，标准差8
- **期末成绩**: 正态分布，平均分72，标准差15

### 成绩查询功能
- 按学号或姓名查询学生
- 显示学生所有科目成绩
- 显示总成绩和平均成绩

### 统计分析功能
- 单个教学班成绩分布
- 所有学生总成绩分布
- 分数段统计(优秀、良好、中等、及格、不及格)
- 可视化柱状图显示

### 排序功能
- 按学号排序
- 按单科成绩排序(升序/降序)
- 按总成绩排序(升序/降序)

- 清晰的职责分工

## 注意事项

1. 系统不使用数据库，所有数据保存在内存中
2. 程序退出后数据会丢失
3. 建议按照菜单顺序依次操作
4. 确保Java版本支持Lambda表达式(Java 8+)
